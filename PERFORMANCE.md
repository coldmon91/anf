# 성능 (Performance)

anf의 속도는 설계의 결과입니다. 아래는 실제 측정값입니다.

## 측정 환경

| 항목 | 값 |
|------|----|
| 칩 | Apple M4 Pro (14 cores) |
| macOS | 26.4.1 (25E253) |
| 빌드 | release (`./build.sh`), Swift + AppKit |
| 측정 폴더 | `kr` — **26,549개** 항목(대부분 한글 폴더명) |

> 측정은 warm cache 기준(반복 실행 평균). 디렉터리 첫 진입(cold)은 커널 디렉터리
> 캐시가 비어 있어 더 느릴 수 있습니다(26,549개 기준 첫 진입 ~0.1초대).

---

## 폴더 진입 (디렉터리 로딩)

핵심은 **`getattrlistbulk(2)` 네이티브 벌크 읽기** — 항목별 `stat` 없이 이름·종류·크기·날짜를
몇 번의 시스템 콜로 한 번에 가져옵니다.

| 26,549개 폴더 | 시간 | 비고 |
|---------------|------|------|
| **getattrlistbulk** (anf, 이름+종류+크기+날짜) | **23.8 ms** | per-item stat 없음 |
| Foundation `contentsOfDirectory` 이름만 | 29.2 ms | 메타데이터 없음인데도 더 느림 |
| Foundation 풀 `resourceValues` (anf의 *옛* 방식) | **1,084.8 ms** | 항목별 stat |

→ 옛 Foundation 풀 stat 대비 **약 46× 빠름**.

작은/보통 폴더(수십~수백 개)는 벌크 읽기 **0.1~0.3 ms**로 사실상 즉시.

## 정렬

한글 이름 수만 개를 정렬할 때, locale 기반 비교는 느립니다. anf는 **소문자 UTF-8 키를
미리 만들어 바이트 사전순**으로 정렬합니다(한글은 유니코드상 사전순이라 결과 동일).

| 26,549개 정렬 | 시간 |
|---------------|------|
| **소문자 UTF-8 키 + 키 생성** (anf) | **21.6 ms** |
| `localizedStandardCompare` (기존 방식) | 108.7 ms |

→ 약 **5× 빠름**. 게다가 정렬은 메인 스레드 밖(백그라운드)에서 수행해 UI를 막지 않습니다.

**합산:** 26,549개 폴더 진입 = 벌크 읽기 ~24 ms + 정렬 ~22 ms ≈ **~46 ms** (warm).
기존 방식(풀 stat 1,085 ms + locale 정렬 109 ms ≈ 1.2초)을 한 번의 패스로 대체.

## 렌더링

파일 목록은 SwiftUI `Table`이 아니라 **네이티브 `NSTableView`**(NSViewRepresentable)로
직접 그립니다. 화면에 보이는 행만 재활용하므로 항목 수와 무관하게 스크롤·정렬·화살표 이동이
즉각적입니다. SwiftUI `Table`은 모든 행 identity를 메인 스레드에서 diff해 수만 개 규모에서
멈춥니다(교체 전 실측: 26,549행에서 1초 이상 정지).

추가로 `FileItem` 생성(자식 URL 구성)은 전 코어로 병렬화해, 26,549개 기준
URL 생성 비용을 **~103 ms → ~16 ms**로 줄였습니다.

## 메모리 · 크기 · 실행

| 항목 | 값 |
|------|----|
| 유휴 메모리(RSS, 홈 폴더) | ~210 MB |
| 바이너리 | 2.97 MB |
| .app 번들 | 3.2 MB |
| 프로세스 기동 | ~0.14 s |

> 메모리에는 내장 터미널용 **WebKit(WKWebView)** 런타임이 포함됩니다(WebKit이 대부분).
> Electron 기반 파일 매니저(보통 300~500 MB+)보다 가볍고, 번들은 3 MB대로 매우 작습니다.

## 검색

- **파일명**: 포커스된 폴더 하위를 `fd`로 백그라운드 인덱싱 → 검색은 인메모리 리스트를
  내장 Swift 퍼지 랭킹으로 즉시 필터링. 인덱스는 checkpoint로 저장돼 재시작에도 즉시 사용.
- **내용**: `ripgrep`으로 본문 검색(메인 스레드 밖, 3초 타임아웃·대용량 파일 스킵).
  `hwpx·docx·pptx·xlsx`는 텍스트 XML만 추출해 Swift에서 매칭(NFC/NFD 정규화 무관).
- 입력은 130 ms 디바운스로 불필요한 작업을 막습니다.
